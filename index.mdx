CozoDB is a general-purpose, transactional database that unifies relational data, graph traversal, and vector search under a single query language: **CozoScript**, a dialect of Datalog. You can embed it directly in your application or run it as a standalone server — with bindings for Rust, Python, Node.js, Java, Swift, Go, C/C++, and the browser (WASM).

## Why CozoDB?

CozoDB brings together capabilities that traditionally required multiple separate databases:

<CardGroup cols={3}>
  <Card title="Relational + Graph" icon="table">
    Store data in relations (tables) and query implicit graph structures without forcing a property-graph model.
  </Card>
  <Card title="Vector Search" icon="magnifying-glass">
    HNSW vector indices integrate natively with Datalog — combine semantic search with recursive graph traversal.
  </Card>
  <Card title="Full-Text Search" icon="file-magnifying-glass">
    Built-in FTS with MinHash-LSH for near-duplicate detection, supporting multiple languages.
  </Card>
  <Card title="Time Travel" icon="clock-rotate-left">
    Query your data at any point in the past with per-relation historical tracking.
  </Card>
  <Card title="Embeddable" icon="microchip">
    Run in the same process as your application — no server needed. Works on mobile and in the browser.
  </Card>
  <Card title="Transactional" icon="shield-check">
    Full MVCC transactions protect against corruption under concurrent writes.
  </Card>
</CardGroup>
